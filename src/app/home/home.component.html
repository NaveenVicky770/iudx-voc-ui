About IUDX Vocabulary

voc.iudx.org.in is a shared vocabulary that details schemas for structured data exchange via IUDX data exchange APIs. The creation and maintenance of voc.iudx.org.in schemas is intended to be a collaborative activity.

A shared vocabulary along with a data exchange standard has the potential to enable data interoperability, functional extensibility and information exchange across silos (application verticals, city departments etc.). Further, shared vocabulary allows intelligent applications and data consumers to understand semantic meaning of attributes and entities entailing the IUDX specification.

The schema used to specify voc.iudx.org.in is derived from the Resource Description Framework (RDF) schema. Apart from standard properties from RDFS, additional properties have been defined to specify vocabulary components. For example, similar to schema.org, additional properties like ‘domainIncludes’ and ‘rangeIncludes’ have been used to allow associating a property with multiple domains and ranges.

About AgriJSON Vocabulary

agrijson.adex.org.in is a shared vocabulary that details schemas for structured agricultural data exchange via data exchange(DX) APIs. The creation and maintenance of agrijson.adex.org.in schemas is intended to be a collaborative activity.

A shared vocabulary along with a data exchange standard has the potential to enable data interoperability, functional extensibility and information exchange across silos (application verticals, departments etc.). Further, shared vocabulary allows intelligent applications and data consumers to understand the semantic meaning of attributes and entities entailing the ADeX specification.

The schema used to specify agrijson.adex.org.in is derived from the Resource Description Framework (RDF) schema. Apart from standard properties from RDFS, additional properties have been defined to specify vocabulary components. For example, similar to schema.org, additional properties like ‘domainIncludes’ and ‘rangeIncludes’ have been used to allow associating a property with multiple domains and ranges.

What does it consist of?

  • Set of ‘types’ (or ‘classes’) representing structured data objects. These ‘types’ are organized into single inheritance hierarchies using the ‘rdfs:subclassOf’ property from RDF Schema

  • Set of ‘properties’ where each property may have one or more types as its domain which implies that this property may be present in the instances belonging to any of the domain types. Further, each property may have one or more types as its range which implies that the value objects for this property should be an instance of at least one of the range types. Additional properties like ‘exactMatch’ may be used to provide additional linkings to already known properties from other vocabularies.

For example, a data model can be represented as a ‘type’ and is associated with all the properties that include this data model type in its domain.

agrijson.adex.org.in uses JSON-LD to represent vocabulary objects.
